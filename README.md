# API Mocker
클라이언트가 서버 구현 전에 기능을 테스트할 수 있도록 API 모킹 기능을 제공하는 서버를 구축한다.

## 목표
- Kotlin의 숙련도를 높인다.
- Framework에 의존하지 않고 간단한 웹 서버를 구현해본다.
  - 기존 프레임워크의 구현을 그대로 따라가기보단, 본인이 생각한 구현안으로 해결하려 시도한다.
- 지금 알고 있는(혹은 이해할 수 있는) 지식만으로 최대한 구현해보고, 앞으로 어떤 지식을 학습해야 할지 필요성을 체감한다.
- 복잡한 시스템을 구현하면서도 SOLID 원칙을 준수하고 단위 테스트를 작성해보는 경험을 한다.

## 기능 요구사항
### 핵심 기능
- [x] 서버를 이용해 클라이언트와 HTTP를 기반으로 통신한다.
   - [x] 계층 아키텍처를 통해, 핵심 비즈니스 로직과 통신 로직을 분리한다.
- [ ] DI를 구현한다.
- [ ] DB 접근을 추상화한다.
- [x] 예외가 발생해도 애플리케이션이 종료되지 않도록 하고, 예외를 전역 처리할 수 있게 한다.
- [ ] watson.yml 파일을 이용해 애플리케이션에 환경변수를 주입한다.

### 추가 기능
- [ ] JWT 기반의 인증/인가를 구현한다.
- [ ] 비밀번호에 대한 단방향 암호화를 구현한다.
- [ ] 라이브러리에 의존한 기능을 직접 구현한다.
